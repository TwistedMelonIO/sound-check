# v1.4.0 Release Notes — Reset OSC Arms All Tracks

**Released: March 18, 2026**

## What's New

### Reset & Start Arm All Track Cues
- **Summary**: Show start and game reset now arm all track cues and disarm WIN/LOOSE in QLab
- **Details**: `armAllTrackCues()` sends staggered OSC commands (10ms apart) to arm every track cue (R1.1 through R6.10) for the current pack and disarm both WIN and LOOSE outcome cues. This fires automatically on `/sound-check/start` and `/sound-check/reset`
- **Impact**: QLab is guaranteed to be in a clean state at the beginning and end of every show — no stale armed/disarmed cues carry over

### Manual Arm-All OSC Command
- **Summary**: New `/sound-check/tracks/arm-all` OSC command
- **Details**: Manually triggers the same arm-all + disarm WIN/LOOSE logic from StreamDeck or Bitfocus Companion
- **Impact**: Operators can reset QLab cue state at any point during a show without a full game reset

## Technical Changes
- All arm/disarm OSC commands use `sendStaggeredOSC()` with 10ms spacing to avoid flooding QLab
- WIN/LOOSE disarm included in the same staggered batch as track arm commands (57 total commands per pack)
- OSC Dictionary updated with new command and updated descriptions for `/sound-check/start` and `/sound-check/reset`
